# Sub Apps

## What Counts As A Sub App

- A sub app lives under `apps/<app-name>/`.
- It can define its own `pages`, `layouts`, and `stores`.
- It is discovered by the existing router and layout plugins. No extra registration is needed for the default case.

## Directory Shape

```text
apps/
  admin/
    layouts/
      default/index.vue
      base/index.vue
    pages/
      index.vue
      user/index.vue
    stores/
      ...
```

## Route Discovery Rules

The router plugin reads:

- global pages from `src/pages/**`
- module pages from `apps/*/pages/**`

For `apps/*/pages/**`, the app folder name becomes the route prefix.

Examples:

- `apps/admin/pages/index.vue` -> `/admin`
- `apps/admin/pages/user/index.vue` -> `/admin/user`

This comes from `plugins/router.ts`, which rewrites:

- `apps/<name>/pages/...`
- to `<name>/...`

## Layout Discovery Rules

The layout plugin reads:

- global layouts from `src/layouts/**`
- module layouts from `apps/*/layouts/**`

Layout keys are generated by:

- removing `.vue`
- removing trailing `/index`

Examples:

- `src/layouts/default/index.vue` -> `default`
- `apps/admin/layouts/default/index.vue` -> `admin/default`
- `apps/admin/layouts/base/index.vue` -> `admin/base`

## Default Module Layout Behavior

For module routes, layout resolution prefers:

1. `meta.layout === false`
2. explicit `meta.layout`
3. `modules[routePrefix].layout`
4. `<moduleName>/default`
5. `modules[routePrefix].fallbackLayout ?? globalFallbackLayout`

With the current repo config:

- `layoutDirs` includes `apps/*/layouts`
- `defaultLayout` is `default`
- `globalFallbackLayout` is `false`

So for `apps/admin/pages/**`, the default layout lookup is:

- `admin/default`

That means the normal sub-app convention is:

- create `apps/<app-name>/pages/**`
- create `apps/<app-name>/layouts/default/index.vue`
- routes under `/app-name/**` will use `<app-name>/default` automatically

## When To Add Module Config

You only need `modules[routePrefix]` config when:

- the route prefix and layout module name should differ
- the default module layout is not `default`
- the module needs a fallback layout

Example:

```ts
modules: {
  admin: {
    module: 'admin',
    layout: 'admin/base',
    fallbackLayout: false,
  },
}
```

## Current Repo Implication

The current `apps/admin` structure already follows this pattern:

- pages under `apps/admin/pages`
- layouts under `apps/admin/layouts`
- default layout at `apps/admin/layouts/default/index.vue`

So `/admin/**` routes are treated as a sub-app and default to `admin/default`.

## Practical Usage

- Use `src/pages` and `src/layouts` for global pages and layouts.
- Use `apps/<app-name>/pages` and `apps/<app-name>/layouts` for isolated module applications.
- Prefer `layouts/default/index.vue` as the default entry layout for each sub-app.
- Add `meta.layout` only when a route needs to opt out or switch away from the sub-app default.
